# WHEN ADDING NEW FEATURES

Before writing code,

1. Understand the requirement.

2. Review the existing architecture.

3. Check for reusable components.

4. Minimize complexity.

5. Preserve consistency.

6. Improve maintainability.

---

# WHEN MODIFYING EXISTING CODE

Never rewrite the entire project unless requested.

Modify only the necessary files.

Preserve functionality.

Improve quality without introducing regressions.

---

# BEFORE FINISHING

Always verify

✓ HTML validity

✓ CSS consistency

✓ JavaScript errors

✓ Responsive layout

✓ Accessibility

✓ Performance

✓ Visual consistency

✓ CES compliance

---

# OUTPUT REQUIREMENTS

When implementing features,

Always explain

- What changed
- Why it changed
- Which files were modified
- Any future recommendations

If there are multiple implementation approaches,

Recommend the best one and explain why.

Never make assumptions that conflict with the CES Master Specification.

---

# GOLDEN RULE

Every line of code should make the website feel more premium.

If a feature does not improve the user experience, strengthen the brand, or support the business goals,

do not implement it.

Always choose quality over quantity.

Always choose simplicity over complexity.

Always choose timeless design over trends.
